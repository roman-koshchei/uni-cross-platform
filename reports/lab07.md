## Постановка задачі

Потрібно створити Java-застосунок для гри "п'ять в ряд". Гравці по черзі ставлять у клітинки хрестики та нулі. Перемагає той, хто першим отримує п'ять власних символів на одній лінії: горизонтально, вертикально або по діагоналі.

Звіт має містити опис алгоритму, який визначає ходи комп'ютера, і пояснення доцільності такого алгоритму.

## Мета

Реалізувати консольну гру "П'ять в ряд" для людини проти комп'ютера, алгоритм ходів комп'ютера, demo-режим і тести.

## Реалізація

У модулі `lab07` використано фіксоване поле `10x10`. Перевірка перемоги виконується в 4 напрямках: горизонталь, вертикаль, головна та побічна діагоналі. Комп'ютер спочатку шукає власний виграшний хід, потім блокує негайний виграш суперника, інакше оцінює клітинки за близькістю до центру, сусідством із фігурами та довжиною потенційної лінії.

Інтерактивний режим використовує `ConsoleHumanStrategy`: користувач бачить поле та вводить рядок і стовпець порожньої клітинки. Demo-режим не потребує вводу з клавіатури: для сторони людини використано окрему автоматичну стратегію, тому запуск завжди завершується сам.

Ключові компоненти:

- `Board` зберігає поле `10x10`, виконує постановку фігур, перевірку п'яти в ряд і оцінку ходів.
- `PlayerStrategy` є спільним інтерфейсом для людини, demo-гравця і комп'ютера.
- `ComputerStrategy` реалізує логіку ходу комп'ютера.
- `ConsoleHumanStrategy` відповідає за інтерактивне введення ходів.
- `HumanStrategy` є простою автоматичною стратегією для demo-режиму.
- `GomokuGame` керує циклом гри до перемоги або нічиєї.
- `Lab07Main` забезпечує інтерактивний або demo-запуск.

## Алгоритм комп'ютера

Алгоритм комп'ютера має три рівні пріоритету:

1. Якщо комп'ютер може виграти поточним ходом, він обирає цей хід.
2. Якщо людина може виграти наступним ходом, комп'ютер блокує цю клітинку.
3. Якщо критичних ходів немає, комп'ютер обирає клітинку з найбільшою оцінкою.

Оцінка враховує близькість до центру, наявність сусідніх власних і чужих фігур, а також довжину потенційної лінії після ходу. Такий підхід не є повним перебором дерева гри, але є доцільним для консольної лабораторної: він швидко працює на полі `10x10`, не пропускає негайні виграші й захищається від очевидної перемоги суперника.

## Програмний код з поясненнями

Фрагмент вибору ходу комп'ютера:

```java
public Move chooseMove(Board board) {
    Move winningMove = findBestCriticalMove(board, Cell.COMPUTER);
    if (winningMove != null) {
        return winningMove;
    }

    Move blockingMove = findBestCriticalMove(board, Cell.HUMAN);
    if (blockingMove != null) {
        return blockingMove;
    }

    return board.emptyCells().stream()
        .max(Comparator
            .comparingInt((Move move) -> board.scoreMove(move, Cell.COMPUTER))
            .thenComparingInt(move -> -Math.abs((Board.SIZE - 1) / 2 - move.row()))
            .thenComparingInt(move -> -Math.abs((Board.SIZE - 1) / 2 - move.col())))
        .orElseThrow(() -> new IllegalStateException("No legal moves left"));
}
```

Метод спочатку перевіряє виграшні та блокувальні ходи. Якщо таких ходів немає, усі порожні клітинки порівнюються за оцінкою, а при рівності перевага віддається клітинкам ближче до центру.

Фрагмент оцінювання ходу:

```java
public int scoreMove(Move move, Cell cell) {
    int center = (SIZE - 1) / 2;
    int score = 40 - Math.abs(center - move.row()) - Math.abs(center - move.col());
    for (int row = move.row() - 1; row <= move.row() + 1; row++) {
        for (int col = move.col() - 1; col <= move.col() + 1; col++) {
            if (!isInside(row, col) || (row == move.row() && col == move.col())) {
                continue;
            }
            if (get(row, col) == cell) {
                score += 12;
            } else if (get(row, col) == cell.opponent()) {
                score += 5;
            }
        }
    }
    Board future = place(move, cell);
    score += future.longestLine(move, cell) * 20;
    return score;
}
```

Цей метод надає перевагу центральним клітинкам, ходам поруч із власними фігурами та ходам, які будують довшу лінію. Додаткові бали за сусідство з фігурами суперника допомагають комп'ютеру не ігнорувати оборонні позиції.

## Тестування

JUnit 5 тести перевіряють:

- знаходження 5 в ряд по горизонталі;
- знаходження 5 в ряд по діагоналі;
- пошук виграшного ходу;
- вибір комп'ютером виграшного ходу;
- блокування комп'ютером виграшного ходу людини.

Команда запуску тестів:

```bash
mvn -q -pl lab07 test
```
