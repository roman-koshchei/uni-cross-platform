## Постановка задачі

Потрібно створити Java-застосунок для гри "Життя" Джона Конвея. Ігрове поле складається з клітинок, кожна з яких може бути живою або мертвою. Кожне наступне покоління формується за правилами:

- жива клітинка з двома або трьома живими сусідами залишається живою;
- жива клітинка з нулем або одним живим сусідом помирає від самотності;
- жива клітинка з чотирма або більше живими сусідами помирає від перенаселення;
- мертва клітинка з рівно трьома живими сусідами оживає.

Роботу застосунку потрібно продемонструвати в різних операційних системах. У цій реалізації підготовлено консольний запуск, який не залежить від конкретної ОС.

## Мета

Реалізувати консольну версію гри "Життя" на скінченному полі без загортання країв і перевірити її роботу тестами.

## Реалізація

Створено Maven-модуль `lab03`, який наслідує кореневий parent `pom`. Програма містить модель поля, рушій для обчислення наступного покоління, завантаження стартового шаблону з рядків або файлу та друк поколінь у консоль. Демонстраційний `main` запускає кілька поколінь і завершується автоматично.

Ключові компоненти:

- `LifeBoard` зберігає стан клітин і рахує живих сусідів без загортання.
- `LifeEngine` обчислює наступне покоління за правилами Conway's Game of Life.
- `PatternLoader` читає стартову конфігурацію з рядків або текстового файлу.
- `LifePrinter` форматує покоління для виводу в консоль.
- `Lab03Application` є демонстраційною точкою входу.

## Програмний код з поясненнями

Фрагмент обчислення наступного покоління:

```java
public LifeBoard nextGeneration(LifeBoard board) {
    boolean[][] next = new boolean[board.rows()][board.cols()];
    for (int row = 0; row < board.rows(); row++) {
        for (int col = 0; col < board.cols(); col++) {
            int neighbors = board.aliveNeighbors(row, col);
            boolean alive = board.isAlive(row, col);
            next[row][col] = neighbors == 3 || (alive && neighbors == 2);
        }
    }
    return new LifeBoard(next);
}
```

Для кожної клітинки рахується кількість живих сусідів. Умова `neighbors == 3` реалізує народження нової клітинки, а `alive && neighbors == 2` залишає живою клітинку з двома сусідами. Випадок трьох сусідів також залишає живу клітинку живою, бо перша частина умови вже істинна.

Фрагмент завантаження стартового шаблону з файлу використовується для запуску довільних конфігурацій:

```java
public LifeBoard fromFile(Path path) throws IOException {
    return fromStrings(Files.readAllLines(path));
}
```

Це відокремлює логіку читання файлу від правил гри: `LifeEngine` не залежить від того, звідки взято початковий стан.

## Тестування

JUnit 5 тести перевіряють:

- еволюцію осцилятора;
- відсутність загортання на краях поля;
- коректне завантаження шаблонів з рядків та файлу.

Команда запуску тестів:

```bash
mvn -q -pl lab03 test
```
