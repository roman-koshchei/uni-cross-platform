## Постановка задачі

Потрібно створити Java-застосунок для гри "Криниця-Ножиці-Папір", де користувач грає проти комп'ютера. Комп'ютер робить хід після вибору користувача, але не використовує поточний хід користувача для шахрайського вибору.

Правила перемоги:

- криниця перемагає ножиці;
- ножиці перемагають папір;
- папір перемагає криницю;
- якщо ходи однакові, раунд переграється.

Застосунок має підтримувати задану користувачем кількість сеансів, три режими комп'ютера, запис результатів у файл і перегляд статистики перемог та поразок.

## Мета

Реалізувати графічну гру "Криниця-Ножиці-Папір" з кількома стратегіями комп'ютера, збереженням історії та статистикою результатів.

## Реалізація

Створено Maven-модуль `lab04`, який наслідує кореневий parent `pom`. Реалізовано правила гри, три стратегії комп'ютера, збереження завершених сеансів у CSV-файл, читання історії та обчислення статистики. Точка входу `Lab04Application` запускає графічний інтерфейс Swing.

У графічному режимі користувач перед початком задає кількість сеансів і обирає режим комп'ютера: випадковий, адаптивний до поточного сеансу або адаптивний до історії з файлу. Ходи виконуються кнопками `Криниця`, `Ножиці`, `Папір`. Якщо ходи однакові, результат вважається нічиєю, сеанс не зараховується як завершений і переграється. У вікні є кнопка перегляду статистики перемог і поразок.

Ключові компоненти:

- `Move` описує ходи й правила визначення переможця.
- `RandomStrategy` виконує випадковий вибір ходу.
- `SessionAdaptiveStrategy` адаптується до ходів гравця в межах поточного сеансу.
- `FileAdaptiveStrategy` адаптується до збереженої історії з CSV-файлу.
- `HistoryStore` записує й читає історію завершених сеансів та статистику.
- `GameSession` проводить раунди без шахрайства зі сторони стратегії.
- `SwingGameFrame` реалізує графічне вікно гри на Swing.
- `Lab04Application` запускає GUI-застосунок.

## Програмний код з поясненнями

Фрагмент визначення переможця:

```java
public Outcome playAgainst(Move other) {
    if (this == other) {
        return Outcome.DRAW;
    }
    if ((this == WELL && other == SCISSORS)
            || (this == SCISSORS && other == PAPER)
            || (this == PAPER && other == WELL)) {
        return Outcome.PLAYER_WIN;
    }
    return Outcome.COMPUTER_WIN;
}
```

Метод порівнює хід гравця з ходом комп'ютера. Спочатку обробляється нічия, потім явно перелічені три виграшні комбінації гравця. Якщо жодна з них не виконалась, перемагає комп'ютер.

Фрагмент запису історії:

```java
public void append(Round round) throws IOException {
    Path parent = path.getParent();
    if (parent != null) {
        Files.createDirectories(parent);
    }
    if (Files.notExists(path)) {
        Files.writeString(path, HEADER + System.lineSeparator());
    }
    String line = round.playerMove().name() + ","
            + round.computerMove().name() + ","
            + round.outcome().name()
            + System.lineSeparator();
    Files.writeString(path, line, StandardOpenOption.APPEND);
}
```

Історія зберігається у CSV-форматі. Перед записом створюється батьківська директорія, а для нового файлу додається заголовок. Ці дані потім використовуються для статистики та адаптивної стратегії, що аналізує попередні сеанси.

Фрагмент запуску графічного інтерфейсу:

```java
public static void main(String[] args) {
    Path historyFile = Path.of(System.getProperty("user.dir"), "lab04-history.csv");
    SwingUtilities.invokeLater(() -> new SwingGameFrame(historyFile).setVisible(true));
}
```

Метод створює шлях до CSV-файлу історії в робочій директорії та відкриває Swing-вікно в Event Dispatch Thread, як рекомендовано для GUI-застосунків Swing.

Фрагмент обробки ходу в GUI:

```java
private void playMove(Move playerMove) {
    Round round = gameSession.playRound(playerMove, gameStrategy);
    appendLog(formatRound(round));
    if (round.outcome() == Outcome.DRAW) {
        statusLabel.setText("Нічия. Сеанс переграється, оберіть знак ще раз.");
    } else {
        completedSessions++;
        refreshStatistics();
    }
}
```

GUI використовує той самий `GameSession`, що й тести. Нічия не збільшує кількість завершених сеансів, а перемога або поразка оновлює статистику з CSV-файлу.

## Тестування

JUnit 5 тести перевіряють:

- правила гри;
- роботу адаптивної стратегії для поточного сеансу;
- адаптацію до історії з файлу;
- коректний підрахунок статистики;
- те, що нічия не записується як завершений сеанс.

Команда запуску тестів:

```bash
mvn -q -pl lab04 test
```
