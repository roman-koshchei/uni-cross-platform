## Постановка задачі

Потрібно створити Java-застосунок з елементами GUI, який виконує конвертацію між системними та несистемними одиницями для таких типів величин: час, відстань, швидкість, маса, площа, температура, тиск, об'єм і енергія.

Застосунок має працювати кросплатформно, тому реалізація побудована на стандартній Java та Swing.

## Мета

Реалізувати кросплатформний Java-застосунок для конвертації величин з окремим ядром обчислень, Swing-інтерфейсом і консольним demo-режимом для headless запуску.

## Реалізація

Модуль `lab02` створено як Maven-модуль, що наслідує кореневий parent `pom`. Логіку конвертації відокремлено від UI: більшість категорій використовують перетворення через базову одиницю з мультиплікативним коефіцієнтом, а температура реалізована окремими афінними формулами.

Підтримано 9 категорій: час, відстань, швидкість, маса, площа, температура, тиск, об'єм, енергія. У кожній категорії доступно щонайменше 3 одиниці, а конвертація працює в обидва боки між будь-якими одиницями однієї категорії.

Ключові компоненти:

- `MeasurementCategory` описує доступні категорії.
- `UnitDefinition` містить одиниці вимірювання та правила перетворення.
- `UnitConverter` виконує переведення між одиницями та перевіряє сумісність категорій.
- `UnitConversionApp` реалізує Swing GUI.
- `UnitConversionCliDemo` демонструє типові конвертації в headless режимі.

## Програмний код з поясненнями

Основний метод конвертації:

```java
public double convert(double value, UnitDefinition from, UnitDefinition to) {
    if (from.category() != to.category()) {
        throw new IllegalArgumentException("Одиниці належать до різних категорій.");
    }
    double baseValue = from.toBase(value);
    return to.fromBase(baseValue);
}
```

Метод не дозволяє змішувати різні категорії, наприклад метри й секунди. Перетворення виконується у два кроки: спочатку значення переводиться в базову одиницю категорії, потім із базової одиниці переводиться в цільову.

Фрагмент опису одиниць температури:

```java
CELSIUS(MeasurementCategory.TEMPERATURE, "Градус Цельсія", "C") {
    @Override
    public double toBase(double value) {
        return value;
    }

    @Override
    public double fromBase(double baseValue) {
        return baseValue;
    }
},
FAHRENHEIT(MeasurementCategory.TEMPERATURE, "Градус Фаренгейта", "F") {
    @Override
    public double toBase(double value) {
        return (value - 32.0) * 5.0 / 9.0;
    }

    @Override
    public double fromBase(double baseValue) {
        return baseValue * 9.0 / 5.0 + 32.0;
    }
}
```

Для температури простого коефіцієнта недостатньо, бо перетворення має зсув. Тому `UnitDefinition` дозволяє окремим елементам enum перевизначати `toBase` і `fromBase`.

## Тестування

Додано JUnit 5 тести, які перевіряють:

- масштабні перетворення між одиницями однієї категорії;
- температурні формули;
- помилку при спробі конвертації між різними категоріями.

Команда запуску тестів:

```bash
mvn -q -pl lab02 test
```

## Запуск

CLI/demo:

```bash
java -cp lab02/target/classes ua.edu.unicrossplatform.lab02.UnitConversionCliDemo
```

GUI:

```bash
java -cp lab02/target/classes ua.edu.unicrossplatform.lab02.UnitConversionApp
```

Для демонстрації на різних ОС передбачено як GUI, так і headless запуск, але без вигадування результатів на ОС, де застосунок фактично не перевірявся.
