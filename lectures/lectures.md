# Конспект лекцій з курсу

Цей файл потрібен як швидка навігація по лекціях і як список тем, які можуть знадобитися під час виконання лабораторних.

## Лекція 00 — Вступ і організація курсу

Ключові тези:
- Лекція переважно організаційна.
- Курс триває 2 семестри: у 5-му залік, у 6-му іспит.
- За семестр можна набрати 60 балів, ще 40 дає іспит.
- Семестр поділено на 2 модулі з контрольними роботами та лабораторними.
- Формат навчання: 1 лекція і 1 лабораторне заняття на тиждень.
- Лекції проходять дистанційно, семестровий контроль відбувається в аудиторіях.

Залежності / технології:
- Організація курсу
- Модульний контроль
- Лабораторні роботи
- Дистанційний формат навчання

Що може знадобитися в лабах:
Технічного матеріалу тут майже немає. Варто пам'ятати формат оцінювання, структуру семестру і те, що лабораторні складають значну частину балів.

## Лекція 01 — Загальні принципи кросплатформного програмування

Ключові тези:
- Кросплатформне програмування дозволяє запускати одну кодову базу на різних ОС і платформах.
- Основна мета підходу: зменшити витрати на розробку і підтримку.
- Досягати кросплатформності можна через мови, фреймворки, веб, контейнеризацію або нативний код з абстракціями.
- Окремо розглянуті Java та C# як типові платформи для кросплатформної розробки.
- Для Java ключовою є компіляція в байткод і виконання через JVM.
- Подальший матеріал курсу сфокусований саме на Java.

Залежності / технології:
- Java, JVM, JavaFX, Spring, Spring Boot, Java EE
- C#, .NET Core, .NET MAUI, Uno Platform, Blazor, Avalonia
- JavaScript, Node.js, Electron
- Python, Rust, Kotlin Multiplatform, SwiftUI
- React Native, Xamarin, Flutter
- Docker, віртуалізація, веб-додатки

Що може знадобитися в лабах:
Це базовий контекст курсу. Для лабораторних важливо розуміти, чому використовується Java і яку роль у кросплатформності відіграють JVM, GUI-бібліотеки та стандартна екосистема.

## Лекція 02 — Знайомство з платформою Java

Ключові тези:
- Java-код з файлів `.java` компілюється `javac` у байткод `.class`.
- Виконання відбувається через JVM, яка може інтерпретувати код або використовувати JIT-компіляцію.
- Мінімальний набір ПЗ: JDK і JRE; типові IDE: IntelliJ IDEA, Eclipse, NetBeans.
- Мінімальна Java-програма складається з класу і методу `public static void main(String[] args)`.
- Показано консольне введення/виведення через `System.out` та `Scanner`.
- Показано базове GUI через `Swing`: `JOptionPane`, `ImageIcon`, `JLabel`, `JPanel`.

Залежності / технології:
- Java, JVM, JDK, JRE, `javac`
- JIT compilation
- IntelliJ IDEA, Eclipse, NetBeans
- `System.out.print`, `System.out.println`
- `Scanner`
- `javax.swing.JOptionPane`, `ImageIcon`, `JLabel`, `JPanel`
- `java.awt.Color`, `Font`, `Dimension`

Що може знадобитися в лабах:
Майже напевно знадобляться базова структура програми, консольне введення/виведення, перетворення рядків у числа і прості діалогові вікна на `Swing`.

## Лекція 03 — Базові типи даних і оператори

Ключові тези:
- Розглянуто примітивні типи: `byte`, `short`, `int`, `long`, `float`, `double`, `char`, `boolean`.
- Пояснено літерали і системи числення: двійкова, вісімкова, шістнадцяткова.
- Автоматичне приведення типів працює не завжди; для деяких випадків потрібне явне приведення.
- Розібрано арифметичні, логічні, порівняльні та побітові оператори.
- Пояснено зсуви, побітові операції й двійкове подання чисел.
- Розглянуто скорочені присвоєння, тернарний оператор і пріоритет операторів.

Залежності / технології:
- Примітивні типи Java
- Явне та автоматичне приведення типів
- Арифметичні оператори, `++`, `--`, `%`
- Логічні оператори: `&&`, `||`, `!`, `^`
- Оператори порівняння: `==`, `!=`, `<`, `<=`, `>`, `>=`
- Побітові оператори: `&`, `|`, `^`, `~`, `<<`, `>>`, `>>>`
- Тернарний оператор `?:`

Що може знадобитися в лабах:
Це одна з найпрактичніших лекцій. Для лабораторних треба впевнено працювати з типами, арифметикою, `%`, умовами, пріоритетом операторів і приведенням типів.

## Лекція 04 — Керуючі інструкції

Ключові тези:
- Для розгалуження логіки використовуються `if` і `if/else`.
- Для вибору з кількох варіантів використовується `switch` із `case`, `break`, `default`.
- Розглянуто цикли `while`, `do-while` і `for`.
- На прикладах показано обчислення суми й інші повторювані алгоритми.
- Розібрано інструкції `break` і `continue`.
- Показано базове перехоплення помилок через `try/catch`.

Залежності / технології:
- `if`, `else`
- `switch`, `case`, `break`, `default`
- `while`, `do-while`, `for`
- `break`, `continue`
- `try`, `catch`, `Exception`
- `Integer.parseInt`, `Double.parseDouble`
- `JOptionPane`

Що може знадобитися в лабах:
Це основа всієї алгоритмічної частини. У лабораторних постійно знадобляться цикли, гілкування і хоча б мінімальна обробка помилок введення.

## Лекція 05 — Масиви

Ключові тези:
- Одновимірні і двовимірні масиви створюються через `new`, а індексація починається з нуля.
- Розмір масиву доступний через `length`.
- Масиви можна ініціалізувати явно, у циклах або випадковими значеннями.
- Для обходу використовуються звичайний `for` і `for-each`.
- Для копіювання, заповнення, сортування і порівняння корисні `Arrays` та `System.arraycopy`.
- Масиви можуть містити як примітиви, так і об'єкти.

Залежності / технології:
- Масиви `[]`
- `for`, `for-each`
- `java.util.Random`
- `java.util.Arrays`
- `System.arraycopy`
- `Object[]`

Що може знадобитися в лабах:
Треба впевнено створювати, заповнювати, обходити та сортувати масиви, особливо двовимірні структури і масиви об'єктів.

## Лекція 06 — Класи та об'єкти

Ключові тези:
- Клас описує поля й методи, а об'єкт створюється через `new`.
- Доступ до полів і методів відбувається через крапку.
- Присвоєння об'єктних змінних копіює посилання, а не сам об'єкт.
- Методи можна перевантажувати, змінюючи набір параметрів.
- Конструктори викликаються під час створення об'єкта й теж можуть бути перевантажені.
- `static`-поля і `static`-методи належать класу, а не екземпляру.

Залежності / технології:
- `class`
- Об'єкти і `new`
- Поля та методи
- Конструктори
- Перевантаження методів
- `public`, `private`, `static`

Що може знадобитися в лабах:
Це база для побудови моделей даних і сервісних класів. У лабораторних часто доведеться правильно проектувати конструктори, інкапсуляцію і межу між статичним та екземплярним станом.

## Лекція 07 — Спадкування

Ключові тези:
- Спадкування реалізується через `extends`.
- У Java дозволене одиничне спадкування класів, але можливе багаторівневе.
- У конструкторах підкласів використовується `super(...)`.
- Важливі модифікатори доступу: `public`, `protected`, package-private, `private`.
- Методи можна переозначати, а `super.method()` дозволяє звернутися до реалізації батьківського класу.
- Змінна суперкласу може посилатися на об'єкт підкласу.

Залежності / технології:
- `extends`
- `super`
- Конструктори
- Переозначення методів
- Поліморфізм
- `final`
- Модифікатори доступу

Що може знадобитися в лабах:
Лекція потрібна для побудови ієрархій класів і спільної поведінки. Варто добре розуміти виклик `super(...)` і різницю між тим, що видно через тип змінної, і тим, що реально є в об'єкті.

## Лекція 08 — Інтерфейси та абстрактні класи

Ключові тези:
- Абстрактний клас може містити і звичайні, і абстрактні методи.
- Об'єкти абстрактного класу створювати не можна.
- Інтерфейс задає контракт, який клас реалізує через `implements`.
- Клас може реалізувати кілька інтерфейсів.
- Інтерфейси можуть містити `default`-методи.
- Через інтерфейсні змінні можна працювати поліморфно.

Залежності / технології:
- `abstract`
- Абстрактні класи
- `interface`
- `implements`
- `default` methods
- Поліморфізм через інтерфейси

Що може знадобитися в лабах:
Це основа для проєктування чистих контрактів між класами. У лабораторних доведеться вибирати між базовим абстрактним класом і інтерфейсом залежно від того, чи потрібен спільний стан або лише спільна поведінка.

## Лекція 09 — Функціональні інтерфейси, лямбда-вирази та посилання на методи

Ключові тези:
- Анонімні класи корисні для одноразових реалізацій інтерфейсів або абстрактних класів.
- Лямбда-вирази дають короткий запис поведінки.
- Лямбди працюють із функціональними інтерфейсами, які мають один абстрактний метод.
- Посилання на методи `::` можуть вказувати на конкретний об'єкт, клас, статичний метод або конструктор.
- Вибір перевантаженого методу через `::` залежить від сигнатури функціонального інтерфейсу.
- Поведінку об'єкта можна змінювати, передаючи іншу лямбду або іншу реалізацію інтерфейсу.

Залежності / технології:
- Анонімні класи
- Функціональні інтерфейси
- Лямбда-вирази `->`
- Method references `::`
- Constructor references

Що може знадобитися в лабах:
Корисно для колбеків, обробників подій, передачі поведінки в методи і скорочення шаблонного коду.

## Лекція 10 — Оброблення винятків

Ключові тези:
- Основний механізм: `try-catch-finally`.
- Ієрархія помилок побудована навколо `Throwable`, `Error`, `Exception`.
- Винятки можна перехоплювати точково за типом.
- Можна використовувати вкладені `try/catch`, перевикидання через `throw` і сигнатури з `throws`.
- Є різниця між `checked` і `unchecked` винятками.
- Можна створювати власні класи винятків.

Залежності / технології:
- `Throwable`, `Error`, `Exception`, `RuntimeException`
- `try`, `catch`, `finally`
- `throw`, `throws`
- `NumberFormatException`
- `ArithmeticException`
- `ArrayIndexOutOfBoundsException`
- `NegativeArraySizeException`

Що може знадобитися в лабах:
У лабораторних часто треба обробляти неправильне введення, індекси, роботу з файлами та власні бізнес-помилки. Тут важливо не ловити все підряд, а перехоплювати саме очікувані типи винятків.

## Лекція 11 — Багатопотокове програмування

Ключові тези:
- Новий потік у Java можна створити через `Thread` або `Runnable`.
- Основні методи керування: `start()`, `run()`, `sleep()`, `join()`, `interrupt()`, `isAlive()`.
- Головний потік теж є об'єктом `Thread`.
- Потоки можуть бути демон-потоками для фонових задач.
- Коли кілька потоків працюють зі спільним станом, потрібна синхронізація.
- Для синхронізації використовується `synchronized`.

Залежності / технології:
- `Thread`
- `Runnable`
- Лямбда-вирази
- `start()`, `sleep()`, `join()`, `interrupt()`, `isAlive()`
- `setDaemon()`
- `currentThread()`
- `setPriority()`
- `synchronized`

Що може знадобитися в лабах:
Корисно для запуску фонових задач і для розуміння потокобезпеки. Якщо лабораторна матиме паралельні обчислення або GUI, без цієї теми легко отримати гонки даних і нестабільну поведінку.

## Лекція 12 — Узагальнені типи даних

Ключові тези:
- `Generics` дозволяють параметризувати класи та методи типами.
- Замість примітивів використовуються класи-обгортки на кшталт `Integer`.
- Узагальнені методи можуть бути як статичними, так і нестатичними.
- Узагальнення сумісні зі спадкуванням.
- Обмеження `extends` звужує допустимі типи.
- Підстановки `?`, `? extends ...`, `? super ...` потрібні для гнучких API.

Залежності / технології:
- `Generics`
- `<>`
- `Integer`, `Character`, `String`
- Generic class
- Generic method
- `?`, `? extends`, `? super`

Що може знадобитися в лабах:
Тема потрібна для типобезпечних контейнерів, колекцій, сервісів і утиліт. Особливо важливо не скочуватися до сирих типів, якщо можна зберегти строгий типізований API.

## Лекція 13 — Стандартні колекції

Ключові тези:
- Колекції Java базуються на інтерфейсах `Collection`, `List`, `Set`, `SortedSet`, `Map`.
- `List` зберігає порядок і допускає дублікати, `Set` не допускає дублювання.
- Основні реалізації: `ArrayList`, `Vector`, `LinkedList`, `HashSet`, `TreeSet`.
- Для безпечного обходу використовуються `Iterator` і `ListIterator`.
- Для кастомного сортування застосовується `Comparator`.
- Клас `Collections` дає готові алгоритми: `sort`, `reverse`, `max`, `min`, `binarySearch`.

Залежності / технології:
- `Collection`, `List`, `Set`, `SortedSet`, `Map`
- `ArrayList`, `Vector`, `LinkedList`, `HashSet`, `TreeSet`
- `Iterator`, `ListIterator`
- `Comparator`
- `Collections`
- `Stack`, `Hashtable`

Що може знадобитися в лабах:
У лабораторних потрібно швидко обирати правильну структуру даних під задачу. Також часто стануть у пригоді сортування, ітератори і власні компаратори.

## Лекція 14 — Створення GUI в Java

Ключові тези:
- Для GUI у Java згадані `AWT`, `Swing` і `JavaFX`, але фокус лекції саме на `Swing`.
- Типовий сценарій: створити компоненти, налаштувати їх, розмістити у контейнері й додати обробники подій.
- Основні компоненти: `JFrame`, `JButton`, `JLabel`, `JTextField`, `JPanel`, `JComboBox`.
- Події обробляються через listener-інтерфейси: `ActionListener`, `MouseListener`, `KeyListener`, `ItemListener`.
- Обробники можна робити анонімними класами, лямбдами або adapter-класами.
- У прикладах використано ручне позиціонування через `setLayout(null)` і `setBounds(...)`.

Залежності / технології:
- `Swing`, `AWT`, `JavaFX`
- `javax.swing`, `java.awt`, `java.awt.event`
- `JFrame`, `JButton`, `JLabel`, `JTextField`, `JPanel`, `JComboBox`
- `ImageIcon`, `BorderFactory`
- `ActionListener`, `MouseListener`, `KeyListener`, `ItemListener`
- `MouseAdapter`, `KeyAdapter`

Що може знадобитися в лабах:
Якщо лабораторна буде з десктопним інтерфейсом, треба вміти швидко зібрати вікно, повісити обробники подій і оновлювати стан компонентів. Це також база для будь-яких завдань на `Swing`.

## Підсумок для лабораторних

Найімовірніше в лабораторних знадобляться такі теми:
- структура Java-проєкту, `main`, JDK/JRE, Maven;
- базові типи, оператори, цикли, умови і масиви;
- класи, об'єкти, спадкування, інтерфейси, абстрактні класи;
- винятки, колекції, узагальнення;
- `Scanner`, `System.out`, парсинг значень;
- базовий `Swing` і оброблення подій;
- багатопотоковість для окремих задач, якщо це буде потрібно умовою лаби.
